## 为什么在有了 Shell 和标准工具链后，你依然需要 `ft`？

在 Linux 或 macOS 下，我们习惯于用 `find` 查找文件，用 `grep` 搜索内容，用 `makefile` 编排任务。然而，当你的工作流跨越到 Windows，或者进入精简的 Docker 容器（如 Alpine 或 BusyBox）时，这些熟悉的“肌肉记忆”往往会因为命令缺失、参数差异或路径规范不统一而失效。

`ft` 的出现，就是为了解决**跨平台文件操作的碎片化**问题。

### 1. 真正的“一次编写，处处运行” (Cross-Platform Consistency)
*   **消除路径分歧：** Windows 的反斜杠 `\` 与盘符，与 Unix-like 的斜杠 `/` 长期以来是跨平台脚本的噩梦。`ft` 内部完全兼容 POSIX 标准，在 Windows 上也能统一使用 `/` 和 `~`，让一套脚本无需修改即可在不同系统间无缝移植。
*   **预编译无依赖：** 不同于需要安装运行时环境的脚本语言（如 Python/Node.js），`ft` 提供针对 Windows, Linux (含 ARM/RISC-V), macOS 的原生二进制文件。即便在最小化的容器环境下，下载即可用，无需担心复杂的依赖链。

### 2. 统一的任务自动化逻辑 (YAML vs Makefile)
*   **易读的编排模式：** `ft` 采用易读的 **YAML** 配置文件。你可以通过 `execute` 调用内置子命令，或通过 `shell` 集成系统原生命令，让任务编排变得像写笔记一样简单，彻底告别 Makefile 晦涩的语法。
*   **强大的变量增强：** 除了系统环境变量，`ft` 引入了**内置动态变量**（如 `$CURDATE`, `$AGODATE1WEEK`）和用户自定义变量。你可以轻松实现“滚动备份文件，并只保留 7 天（自动删除 7 天前的旧档）”这种在传统 Shell 中需要复杂日期计算和逻辑判断的任务。

### 3. 智能化的属性筛选 (Beyond Match Names)
*   **多维度过滤：** 标准 `ls` 或 `find` 在处理文件属性时语法各异且难以记忆。`ft` 将 `size`（支持 KB/MB/GB 单位）、`time`（支持 “1 month ago” 易读格式）和 `MIME Type` 集成到全局参数中。
*   **MIME 深度集成：** 无需复杂的魔数检查，直接通过 `--mime_includes='image/'` 就能锁定所有图片文件，甚至可以自定义 MIME 映射。

### 4. 强大的“快捷应用”集合 (Built-in Superpowers)
`ft` 内置了许多标准工具链中需要多个命令组合才能实现的高级功能：
*   **`mirror`：** 智能增量同步，仅处理变更部分，是构建备份系统的理想选择。
*   **`erase`：** 符合安全标准的多次覆写擦除，确保敏感数据不可恢复。
*   **`fdups`：** 快速检测并清理海量数据中的重复文件，节省存储空间。
*   **`search`：** 提供跨平台一致的正则匹配引擎，不再受限于不同版本 `grep` 的差异；同时，它支持**仅输出正则表达式捕获的内容**，方便将结果传递给其他程序进一步处理。

### 5. 高增值的知识复用
通过使用 `ft` 及其配套的**[《快速上手简明手册》](started.md)**，你不仅是在完成工作，更是在潜移默化中掌握一系列通用的计算机核心概念：
*   **Glob 通配符：** 掌握现代开发中主流的路径匹配逻辑。
*   **Regex 正则表达式：** 掌握跨语言通用的文本处理核武器。
*   **MIME & Stat：** 深入理解**文件类型**以及文件在系统中的**底层元数据**。

### 6. 从 CLI 到 GUI 的全栈生态
`ft` 是专业文件管理应用 **FileShows** 的核心引擎。这意味着你在命令行中沉淀的自动化配置（YAML），未来可以无缝衔接至具备图形界面的 Windows, macOS, Android 甚至 iOS 端的 `FileShows` 应用中，实现生产力的全维度覆盖。
